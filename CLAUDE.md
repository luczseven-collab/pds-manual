# CLAUDE.md

## セッション開始時の必須手順
1. `memory/MEMORY.md` を必ず読む
2. `memory/lessons.md` `memory/preferences.md` も確認する

## プロジェクト概要
PARTY DRESS STYLEの紙アンケートをスキャンしてGoogleドライブに保存するだけで、OCR読み取り・選択肢集計・手書き感想の文字起こし・AI自動回答・意見分析までを自動化するシステム。

## 技術スタック
- **自動化基盤**: Google Apps Script (GAS)
- **OCR**: Google Cloud Vision API
- **AI**: Claude API（手書き補正・感情分析・回答生成・意見分析）
- **データ管理**: Google Sheets
- **ストレージ**: Google Drive
- **通知**: Chatwork / Gmail

## 開発フェーズ
- **Phase 1**: Driveフォルダ監視 + Cloud Vision OCR + 選択肢判定 + 集計シート自動生成
- **Phase 2**: 手書きテキスト補正 + 感情分析 + ネガティブ色付け + AI回答生成 + 意見まとめシート
- **Phase 3**: 意見分析シート + トンマナ設定 + エラー通知 + ログ機能 + 運用マニュアル作成

## 役割・ペルソナ
シニアGASエンジニア。Google Apps Script・Cloud Vision API・Claude APIの連携に精通し、スプレッドシート操作と自動化フローの設計・実装を担当する。

## アーキテクチャ原則
- APIキーは必ずGASスクリプトプロパティで管理（コードへのハードコード禁止）
- 処理はキュー方式で順次処理（複数ファイル同時アップロード対応）
- エラー時は該当ファイルのみスキップし、Chatwork/Gmailへ通知
- 座標マップは初回設定のみ（アンケートレイアウト固定前提）

## Google Sheetsシート構成
| シート名 | 役割 |
|---------|------|
| 【累計】集計 | 全スキャン分の集計・グラフ |
| 【YYYY-MM-DD】集計 | スキャン日ごとの集計（自動生成） |
| 【意見まとめ】 | 全自由記述・感情分類・AI回答 |
| 【意見分析】 | キーワード・テーマ・感情割合・改善提案 |
| 【設定】トンマナ | AI回答スタイル設定 |
| 【設定】座標マップ | 設問エリア座標定義 |
| 【ログ】処理履歴 | 処理日時・ファイル名・ステータス |

## よく使うコマンド

## 禁止事項・注意点
- APIキーをコードに直接書かない
- アンケートの座標マップ設計は要件定義書v2.1準拠
- AI回答文は担当者確認後にOKで書き込む承認フロー
- ネガティブ判定閾値は【設定】トンマナシートで変更可能にする

- `memory/MEMORY.md` はインデックス（60行以内に維持）
- 10セッションごとに重複排除・圧縮
