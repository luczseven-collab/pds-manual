# アーキテクチャ設計

## 全体構成

```
Google Drive（監視フォルダ）
    ↓ ファイル検知（GASタイマートリガー 5〜10分）
Google Apps Script（メイン処理）
    ├── Cloud Vision API（OCR・座標解析）
    ├── Claude API（補正・感情分析・回答生成）
    └── Google Sheets（集計・意見・ログ）
         → Chatwork / Gmail（通知）
```

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `Code.gs` | メインエントリ・タイマートリガーハンドラ・処理キュー管理 |
| `OcrUtils.gs` | Cloud Vision API呼び出し・テキスト/座標抽出 |
| `CoordMapper.gs` | 座標マッピング・○印判定ロジック |
| `SheetsUtils.gs` | Sheetsへの書き込み・シート自動生成 |
| `AiUtils.gs` | Claude API連携（補正・感情分析・回答生成・意見分析） |
| `NotifyUtils.gs` | Chatwork/Gmail通知 |
| `Config.gs` | スクリプトプロパティ読み書き・設定管理 |

## 処理フロー詳細

### Phase 1: OCR・集計
1. タイマートリガーで監視フォルダの新規ファイルを検知
2. Cloud Vision APIでOCR実行（全文テキスト + 座標情報取得）
3. 【設定】座標マップシートの定義と照合し○印の選択肢を判定
4. 【累計】集計シートと【YYYY-MM-DD】集計シートへ書き込み

### Phase 2: AI処理
5. 手書きテキスト（身長・エリア・自由意見等）をClaude APIで補正
6. 自由意見をポジティブ/ニュートラル/ネガティブに感情分類
7. ネガティブ行をセル背景色 `#FFE0E0` でハイライト
8. トンマナ設定に基づきAI回答文を生成（承認フロー）
9. 【意見まとめ】シートへ集約

### Phase 3: 分析・通知
10. 【意見分析】シートをAIで自動更新（頻出KW・テーマ・感情割合・改善提案）
11. 処理完了をChatworkへ通知
12. エラー時はGmail + Chatworkへ通知
13. 【ログ】処理履歴シートへ記録

## セキュリティ
- APIキーはGASスクリプトプロパティで管理（コードへのハードコード禁止）
- 元スキャン画像は90日後に自動削除

## 制約・前提
- アンケートレイアウト固定（座標マップは初回設定のみ）
- PDF/JPEG形式のみ対応（複合機スキャン出力）
- 1枚あたり処理時間：3分以内
- 同時処理：キュー方式で順次処理
